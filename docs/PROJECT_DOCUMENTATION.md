# Web宠物记忆项目文档

## 1. 项目概述

- 项目名称：Pet Memorial Scrapbook（宠物数字纪念册）
- 项目定位：宠物档案与回忆沉淀平台，兼顾私密记录、公开展示与社区互动
- 当前阶段：毕业设计增强版已落地并完成构建/测试验证

## 2. 核心能力总览

- 用户认证与权限隔离
  - 注册、登录、获取当前用户
  - JWT 无状态鉴权
  - 登录失败精确提示（账户不存在/密码错误）
  - 路由守卫与 401 自动清理登录态
- 宠物档案管理
  - 新增、编辑、删除、详情、列表
  - 性别字段使用选项输入，保证数据一致性
  - 公开开关与分享 token
- 回忆时间线
  - 图文视频回忆（标题/正文/日期/地点）
  - 视频字段：videoUrl、videoCoverUrl、videoDurationSeconds
  - 时间线展示支持图片直显与视频播放
- 媒体上传
  - `/api/files/upload`：图片+视频
  - `/api/files/upload-image`：仅图片
  - 单文件大小上限 1024MB
- 公开访问
  - Token 分享页：`/share/:token`
  - 用户主页后缀页：`/home/:ownerUsername?pet=...`
- 宠物微社区
  - 动态发布（弹窗）
  - 点赞、评论、纪念接力
  - 搜索（标题/正文/宠物名/用户名）
  - 推荐内容（自定义评分）
  - 宠物热榜（聚合互动热度）

## 3. 技术架构

### 3.1 后端

- Spring Boot 3.3.5
- Spring Security + JWT
- Spring Data JPA
- H2（默认）/MySQL（可选）

关键分层：

- controller：接口层
- service：业务逻辑
- repository：数据访问
- entity/dto：模型定义

### 3.2 前端

- Vue 3 + TypeScript
- Vite 7
- Element Plus
- Pinia + Vue Router
- Axios（统一拦截）

关键模块：

- views：登录、主页、宠物详情、社区、公开页
- api：按业务分离请求封装
- types：接口类型定义
- stores：用户会话状态

## 4. 重点创新实现

### 4.1 社区推荐算法

在社区服务层对帖子计算 `recommendationScore`，综合：

- 互动分：点赞数、评论数
- 时效分：随发布时间衰减
- 叙事创新加分：宠物第一视角、纪念接力、叙事模式
- 情绪标签加分：moodTag 差异化加成

### 4.2 宠物热榜

按宠物聚合其帖子互动指标，输出：

- 发帖数
- 总点赞/总评论
- 热度分 `heatScore`

用于前端榜单展示，形成社区活跃可视化。

### 4.3 视频增强链路

- 上传：支持 MP4/WebM/MOV
- 前端：上传后自动读取视频时长
- 前端：自动截取首帧并上传作为预览图
- 展示：社区/回忆/公开页都支持视频播放与时长标识

### 4.4 公开主页后缀

从单一 token 分享扩展为用户维度公开空间：

- 后端接口：`GET /api/public/users/{ownerUsername}/home?pet=`
- 前端路由：`/home/:ownerUsername?pet=shareToken`
- 支持同一用户多宠物公开内容切换

## 5. 运行与部署说明

### 5.1 本地开发

- 推荐脚本：
  - 启动：`scripts/start-dev.ps1`
  - 停止：`scripts/stop-dev.ps1`
- 端口：
  - 后端：8080
  - 前端：3000

### 5.2 云端映射（可选）

- 支持 FRP 将前端映射到云端端口（示例：25565）
- 启动脚本支持联动 frpc 进程
