# Web宠物记忆项目需求说明（Requirements）

## 1. 文档目标

本需求文档用于定义宠物数字纪念册系统的功能边界、验收标准与非功能约束，覆盖当前毕业设计增强版本。

## 2. 项目范围

### 2.1 In Scope（本期范围）

- 用户认证与会话管理（注册/登录/JWT）
- 宠物档案管理（CRUD + 公开开关）
- 回忆管理（图文视频时间线）
- 媒体上传（图片/视频）
- 公开访问页（token 分享 + 用户主页后缀）
- 宠物微社区（发布、评论、点赞、搜索、推荐、热榜）

### 2.2 Out of Scope（本期不包含）

- 多租户企业权限模型
- 原生移动端 App
- AI 自动文案生成
- 复杂关注关系/私信系统

## 3. 角色定义

- 游客：可访问公开分享页和用户公开主页
- 注册用户：可管理自己的宠物、回忆、社区内容
- 管理员：本期不启用专用后台角色

## 4. 功能需求

### 4.1 FR-001 用户认证

- 用户可注册账号并自动登录
- 用户可通过用户名+密码登录
- 登录失败需给出明确提示（账户不存在/密码错误）
- 未登录访问受保护页面需跳转登录页

验收标准：

- 返回统一结构 `{ code, message, data }`
- 登录成功返回 token、用户名、邮箱、角色
- 登录失败提示可区分账户问题与密码问题

### 4.2 FR-002 宠物档案管理

- 用户可创建/编辑/删除宠物
- 支持字段：昵称、物种、品种、性别、生日、纪念日、头像、简介、公开开关
- 性别字段应使用选项输入（如：雄性/雌性/未知），避免自由文本造成统计偏差
- 宠物列表仅展示当前用户数据

验收标准：

- 必填字段校验生效
- 删除宠物时其关联回忆同步删除
- 上传头像后在“我的宠物”列表中可直接显示图片
- 性别字段以预设选项渲染并可正确保存

### 4.3 FR-003 回忆时间线（图文视频）

- 用户可为宠物新增/编辑/删除回忆条目
- 回忆支持字段：标题、正文、日期、地点、图片、视频、视频封面、视频时长
- 回忆按时间倒序展示

验收标准：

- 回忆图片以图片形式直接展示（非纯链接）
- 回忆视频可播放，支持封面与时长显示
- 单条回忆仅所属用户可操作

### 4.4 FR-004 媒体上传

- 支持图片格式：JPG/PNG/WebP
- 支持视频格式：MP4/WebM/MOV
- 单文件大小不超过 1024MB

验收标准：

- 非法格式与超限上传应返回明确错误
- 上传成功后可用于回忆或社区内容发布

### 4.5 FR-005 公开访问能力

- 支持 token 分享页：`/share/:token`
- 支持用户主页后缀：`/home/:ownerUsername?pet=shareToken`
- 用户主页支持切换该用户公开宠物

验收标准：

- 未公开宠物不可被公开页面访问
- 同一用户可在主页中切换多只公开宠物
- 复制链接可生成带用户名后缀地址

### 4.6 FR-006 宠物微社区

- 发布动态（弹窗方式）
- 支持图文视频帖子
- 支持评论、点赞、纪念接力
- 支持按 mode/mood/keyword 搜索
- 提供推荐内容与宠物热榜

验收标准：

- 发布成功后 feed 可即时更新
- 搜索可匹配标题、正文、宠物名、用户名
- 推荐栏与热榜栏可展示有效数据
- 社区中图片应直接显示，视频应显示时长

### 4.7 FR-007 前端体验与视觉优化

- 登录页应在失败时提供可理解的精确提示
- 核心页面（主页、社区、公开页）应具备统一视觉风格与清晰层次
- 动态发布交互应采用弹窗，避免挤压主阅读区域

验收标准：

- 登录错误提示能区分“账户不存在”和“密码错误”
- 社区发帖、浏览、搜索、推荐、热榜流程在一个页面内顺畅完成
- 页面在桌面端和移动端均可正常加载与操作

## 5. 非功能需求

### 5.1 性能

- 常规列表查询平均响应时间目标 < 500ms（开发环境除外）
- 前端页面切换与交互无明显卡顿

### 5.2 可用性

- 核心流程（登录→建宠物→建回忆→发布社区）应短路径完成
- 错误提示清晰、可理解、可恢复

### 5.3 安全

- 受保护接口需要 JWT
- 用户数据隔离：不可访问他人私有数据
- JWT 密钥、数据库账号在生产环境使用环境变量

### 5.4 可维护性

- 前端按 views/api/stores/types 分层
- 后端按 controller/service/repository 分层
- 关键链路具备基础自动化测试

## 6. 接口与数据约定

### 6.1 统一返回格式

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 6.2 API 约定

- 前缀：`/api`
- 鉴权头：`Authorization: Bearer <token>`

### 6.3 核心实体

- User
- Pet
- MemoryEntry
- CommunityPost
- CommunityComment
- CommunityPostLike

## 7. 测试需求

- 后端：服务层与鉴权链路测试
- 前端：关键页面渲染与构建检查
- 交付前至少通过：
  - `mvnw.cmd test`
  - `npm run build`
  - `npm run test:unit -- --run`

## 8. 优先级

- P0：认证、宠物/回忆 CRUD、媒体上传
- P1：公开主页后缀、社区弹窗发布、搜索
- P2：推荐算法、热榜、体验优化

## 9. 当前实现映射

- 已完成：P0 + P1 + P2 核心能力
- 后续迭代：分页、对象存储、CI/CD、推荐策略参数化
